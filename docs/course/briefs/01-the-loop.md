# Module 1: The Loop — A Team of Specialists That Never Meet

Module file to write: `modules/01-the-loop.html` — a single `<section class="module" id="module-1">` block (odd module: default background). ~5 screens.

## Course-wide context (same in every brief)

- **Course:** "Inside praxis — Prompts, Gates & Handoffs" — teaches the praxis repo, a Claude Code **plugin marketplace**: five plugins (research, grounding-wiki, educate, build, codebase-to-course) that form a research → teach → build loop on a shared chassis of tiny Node.js modules (`lib/`). Plugins never call each other — they compose only through files on disk and automated checks called gates.
- **Learner:** a "vibe coder" — builds software by instructing AI tools in natural language, zero CS background assumed. Tooltip every technical term aggressively (plugin, repo, Markdown, JSON, manifest, CLI, Node.js, open source…). Tone: smart friend, never professor.
- **Accent color:** teal. Refer to plugins by their lowercase names (educate, build…). The AI assistant is "Claude" / "Claude Code" (tooltip Claude Code on first use: a tool where an AI works on your files from the terminal).
- **True meta-fact, use it:** THIS very course was generated by the `codebase-to-course` plugin that lives inside the codebase it's teaching, its output landed in `docs/course/` exactly where the plugin's docs say it should, and it had to pass the repo's own automated quality gate before you ever saw it.

## Teaching Arc

- **Metaphor:** a natural-history museum staffed by specialists who never meet. Field researchers collect specimens (research), archivists label each specimen with exactly when and where it was verified (grounding-wiki), educators run the workshops (educate), the workshop crew builds the hands-on exhibits from the educators' written orders (build), and exhibit designers turn the whole collection into the public gallery (codebase-to-course). None of them ever talk face-to-face — everything moves through labeled trays and paperwork. That "coordination through paperwork, not meetings" is the whole architecture of praxis.
- **Opening hook:** "The course you're reading right now was built by the code it's about to teach you." Then: you've used AI coding tools; praxis is what it looks like when someone builds *apps for the AI itself* — installable skills that teach Claude new jobs.
- **Key insight:** praxis isn't an app with buttons — it's a suite of five installable AI skills that cooperate *without ever calling each other*, passing work through files like museum departments passing trays.
- **Why should I care?** This is the architecture pattern for making AI work *reliable*: split big jobs into single-purpose phases, connect them with files you can read, and check every claim automatically. You can steal this shape for any AI workflow you build.
