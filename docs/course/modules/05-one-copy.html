One Copy Everywhere

Why praxis deliberately copies its shared code into all five plugins — and the robots that make copying safe

The rule we're about to break

Every programmer is taught the same commandment — DRY: never copy-paste code, share it. Praxis copies its shared code into all five plugins. On purpose. And it's the right call.

At the depot — development
🚚🚚🚚🚚🚚
⤵  one extension cord each  ⤵
⚡
lib/ — the depot's power supply

One copy of the shared code. Every plugin plugs into it.

On the road — shipped
🚚⚡ 🚚⚡ 🚚⚡
🚚⚡ 🚚⚡
five trucks, five generators

Every shipped plugin carries its own full copy of lib/ inside it.

Picture the plugins as a food-truck fleet. During development all five trucks park at one depot and plug into the depot's power — the repo-root chassis. But a truck that ships out can't drag an extension cord back to the depot — so at packaging time, each truck gets its own generator installed.

Vendoring: install the generator

The packaging loop in the build script does three things per truck: copy the plugin, install the generator, fix the paths. Notice the dist/ folder is pure output — nobody edits it by hand.

CODE — scripts/build.mjs 

for (const plugin of targets) {
  const src = join(repo, plugin), out = join(dist, plugin);
  if (!existsSync(src)) { console.error(`no such plugin: ${plugin}`); process.exit(1); }
  cpSync(src, out, { recursive: true });                 // plugin sources
  cpSync(join(repo, "lib"), join(out, "lib"), { recursive: true }); // vendor the chassis
  rewriteLibImports(out);                                // ../../lib → ../lib
  console.log(`packaged ${plugin} → dist/${plugin} (lib vendored, imports rewritten)`);
}
PLAIN ENGLISH

For each plugin we're shipping — each truck rolling out…

…note where its source lives and where its packaged copy will go.

Copy the entire plugin folder into the shipping area.

Fix every "where's the shared code?" path, because lib/ now lives one level closer.

Announce what just happened, for the human watching the build.

…and move on to the next truck.

That path-fixing step matters. In development, an import says "go up two folders to find lib" — but inside the packaged truck, lib/ is one folder away. A find-and-replace, performed by code, so humans never maintain two versions of the paths.

CODE — scripts/build.mjs 

function rewriteLibImports(dir) {
  for (const e of readdirSync(dir, { withFileTypes: true })) {
    const p = join(dir, e.name);
    if (e.isDirectory()) { if (e.name !== "lib") rewriteLibImports(p); continue; }
    if (extname(p) === ".mjs") {
      const s = readFileSync(p, "utf8");
      const r = s.replaceAll("../../lib/", "../lib/");
      if (r !== s) writeFileSync(p, r);
    }
  }
}
PLAIN ENGLISH

A helper that fixes the import paths in one folder.

Look at every item inside that folder…

…and work out each item's full location.

If it's a folder, dive in and fix its files too — but skip the freshly copied lib/ itself.

Only touch code files — the .mjs ones.

Read the file's text…

…replace every "up two folders to lib" with "up one"…

…and only rewrite the file if something actually changed.

…close the code-file check.

…close the loop over the folder's contents.

Done — every path now points at the truck's own generator.

The nightly binder check

Some shared content can't be imported at all. A planted template is a standalone file — at runtime it can't reach back into lib/. So the shared blocks live as literal photocopies inside the consumer files — and photocopies drift. Praxis handles it like the fleet's master recipe binder:

1
Mark it

The master and every copy carry the same "start"/"end" marker lines around each shared block.

2
Stamp it

A script — sync-shared.mjs — photocopies each marked region from the canonical source into every consumer file.

3
Compare it, every night

A test cuts the block out of the master and out of each copy and compares them, byte for byte.

4
Fail loudly

Drift fails the pre-commit hook, naming the file and the fix — before the truck ever leaves.

The photocopy list itself is plain data — what gets stamped where:

CODE — scripts/sync-shared.mjs 

export const SYNCS = [
  { source: "lib/html/base.html",
    regions: ["praxis:tokens", "praxis:theme"],
    consumers: ["educate/templates/.template/deck.html"] },
  { source: "lib/toolkit/tooltip.md",
    regions: ["praxis:tooltip-css", "praxis:tooltip-js"],
    consumers: ["educate/templates/.template/deck.html"] },
];

And here's the robot's scissors — the function that cuts a marked block out of any file so master and copy can be compared. It uses a regular expression:

CODE — scripts/sync-shared.mjs 

/** The text between the `<name>:start` and `<name>:end` marker lines (markers excluded). */
export function extractRegion(text, name) {
  const re = new RegExp(`^.*${name}:start.*$\\n([\\s\\S]*?)^.*${name}:end.*$`, "m");
  const m = text.match(re);
  return m ? m[1] : null;
}
PLAIN ENGLISH

The developer's own note: grab what sits between the start and end markers.

Given a file's full text and a region name…

…build a search pattern: find the ":start" line, everything after it, then the ":end" line.

Run that search on the file.

Found it? Hand back just the middle — the shared block. Not found? Say so honestly.

Done — now the master and every copy can be cut out and compared exactly.

If you hand-edit a copy, the comparison fails — and the failure message names the file and the fix: run scripts/sync-shared.mjs. The check also runs as a regular test, so the pre-commit suite fails on drift too.

💡
The sin isn't duplication — it's unchecked duplication

"Never copy-paste" is really "never copy-paste without a robot watching." Share the source of truth, duplicate the shipped artifact, and pay for it with automation that makes drift impossible to miss.

Self-contained all the way down

The same philosophy applies to the HTML pages the plugins produce. They're published to a host whose security policy blocks all outside connections — no CDN scripts, no remote fonts. So a shared checker scans every page for anything that phones home:

🪆
Same philosophy, three layers

Plugins ship self-contained, shared regions are robot-stamped copies, and even the HTML output must carry everything it needs. One rule, all the way down: never depend on something that might not be there.

The binder has recipes, not just wiring

It's not only code that rides along — shared content does too. lib/toolkit/ holds teaching material the plugins borrow, under one strict house rule:

📚

Content, not just code

Pedagogy rules, tooltip snippets, diagram guides — written once in lib/toolkit/, borrowed by any plugin's skills.

🧾

Every borrow states a fallback

A skill that uses a toolkit module must say, in writing, what it does when that module isn't there.

🪂

Graceful degradation

Graceful degradation: the skill still works without the borrowed piece — a bit plainer, never broken.
