Apps for your AI

praxis is a marketplace of plugins, and each plugin carries a skill. Three words unlock everything here:

🧩
Plugin

An add-on you install once. From then on, Claude permanently knows a new job — no new app, no new website.

📜
Skill

The playbook inside the plugin — the step-by-step instructions Claude follows when you ask for that job.

🏪
Marketplace

The store shelf holding the plugins. praxis is both the shelf and the five things on it.

Installing the whole museum staff takes six slash commands — no download, no setup wizard:

/plugin marketplace add ~/Claude/Code/praxis
/plugin install research@praxis
/plugin install grounding-wiki@praxis
/plugin install educate@praxis
/plugin install build@praxis
/plugin install codebase-to-course@praxis

Meet the museum staff

Picture a natural-history museum staffed by five specialists who never meet. Everything moves between departments on labeled trays and paperwork — nobody taps anybody on the shoulder.

🔎

research

Field researcher

Collects specimens from the open web: cited facts on any topic, filed as notes future-you can trust.

Runs in: a notes folder on your machine

🏷️

grounding-wiki

Archivist

Labels every claim about a codebase with exactly where and when it was verified, building a corpus.

Runs in: any repo's wiki folder

🎓

educate

Educator

Runs Socratic lessons, then writes a SPEC for the workshop.

Runs in: a learning folder you pick

🔨

build

Workshop crew

Builds the hands-on exhibit exactly as the SPEC orders, verifies it works, and reports findings back.

Runs in: whatever project the SPEC targets

🖼️

codebase-to-course

Exhibit designer

Turns a whole codebase into the public gallery — an interactive course like the one you're reading.

Runs in: any repo → its docs/course/

Each specialist is one plugin, independently installable. None of them ever calls another — hold onto that, because it's the whole trick.

The loop: coordination by paperwork

So how do specialists who never meet get anything done together? Every hand-off is a file left on a tray — Markdown notes, a SPEC, a findings report. Here's the main loop:

1

research + grounding-wiki stock the trays with verified notes

→
2

educate teaches you, then writes a SPEC

→
3

build implements the SPEC in real code

→
4

findings flow back, and educate revises the lesson

And one side gallery: the archivist's corpus becomes the public exhibit.

1

grounding-wiki maps a codebase into a corpus

→
2

codebase-to-course reads the corpus and the code

→
3

an interactive course lands in docs/course/ — this page

The repo's own README sketches the same loop in four lines:

research (topics) ─┐
                   ├─grounding─▶ educate ──SPEC──▶ build ──findings──▶ educate (revise)
grounding-wiki ────┘             (teach)          (implement)          (fold in)
   (codebase)  └──corpus──▶ codebase-to-course (interactive course in docs/course/)

How a tray actually travels (the handoff) is Module 4's story, and the automated quality checks (gates) are Module 3's — for now, just know both exist.

The whole "app store" is one small file

Remember the marketplace from earlier? Its entire listing is a manifest: one small JSON file. The plugins share a common toolbox of Node.js code, and this file is the store's front door:

CODE 

{
  "name": "praxis",
  "description": "A suite of composable knowledge-work plugins (research, educate, build) on a shared Node chassis.",
  "version": "0.1.0",
  "plugins": [
    {
      "name": "educate",
      "source": "./educate",
      "description": "Turn a folder into a Socratic learning project: teach, author a build SPEC, and gate each lesson done on auditable artifacts."
    }
  ]
}
PLAIN ENGLISH

Open the shipping label — everything the store knows lives between these braces.

A version number, so Claude Code can tell an old listing from a new one.

Now the shelf of plugins begins…

Its one-line pitch: teach lessons, write a SPEC, and only count a lesson done when there's proof.

Close educate's card…

…close the shelf…

…close the label. That's the entire app store listing.
