TASK-27 — Bookkeeping Is Not Thinking
Module 1 · The $0 Round

Stop paying a genius to file paperwork

The pipeline's early runs had a quiet inefficiency: whenever the gate failed because the task board was merely out of date — pure filing, no thinking — the workflow summoned a full AI session to fix it. About 60 cents and a minute of world-class intelligence, spent ticking boxes a program had already computed the exact fixes for.

🧾
The tell

The planner's output IS the fix: a list of ready-to-run commands. When the remedy is already written down, executing it is clerical work — and clerks should be free.

The new station, verbatim

This code was written by the pipeline itself (more on that in module 2). Note the two safety choices — the refusal rail and the honest ending:

CODE 

if (!line.startsWith("backlog task edit ")) throw new Error(`reconcile ${runId}: plan emitted a non-command line, refusing to execute: ${line}`);
commands.push(line);
// Honest report, no retry: a non-empty re-plan means this failure wasn't pure bookkeeping.
const planEmpty = replan.status === 0 && replan.stdout.trim() === "";
PLAIN ENGLISH

Only one shape of command may ever be executed — anything unexpected from the planner is treated as a bug and refused, not run.

Everything that passes the rail joins the to-run list.

After running the fixes, ask the planner again — and if work remains, say so instead of looping.

"Fully reconciled" is only claimed when the planner has nothing left to say.

Key Insight

Escalation ladders are how you spend intelligence well: cheapest fixer first, and each rung must honestly report what it couldn't fix rather than retry forever. The model becomes the escalation, not the default.

Module 2 · Born of the Pipeline

This feature wrote itself. Literally.

Everything module 1 taught you was implemented by the pipeline it improves. The feature was written as a spec, put on the project's own task board, and handed to the pipeline: an AI session in an isolated copy of the repository, judged by the repository's own gates, landing as a normal pull request that a human reviewed. Self-hosting, run one.

It was not a smooth ride — and the bumps taught more than smoothness would have:

1
Killed at minute ten. Twice.

The runner's session time limit — sized for quick fixes — executed the agent mid-work, twice. What saved the run: it had been told to commit every working slice immediately. Death lost nothing that mattered.

2
It found a real bug nobody knew about

Inside its isolated copy, the project's pre-commit checks were quietly leaking the repository's identity into test sandboxes — test repos could touch the real one. The agent diagnosed it and patched the root cause, unprompted. (One piece of earlier collateral — a flipped config flag — needed a human to spot.)

3
It obeyed house rules nobody recited

The repo's conventions demand documentation stay in sync with code. The agent read the house manual in its checkout and ran the full documentation ritual — unprompted, correctly.

Key Insight

Self-hosting worked not because the AI was flawless — it was killed twice — but because the structure forgave failure: commit-per-slice made death cheap, gates made claims checkable, isolation made mistakes local, and a human held the only pen that signs.
