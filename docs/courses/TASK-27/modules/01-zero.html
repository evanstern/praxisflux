Module 1 · The $0 Round

Stop paying a genius to file paperwork

The pipeline's early runs had a quiet inefficiency: whenever the gate failed because the task board was merely out of date — pure filing, no thinking — the workflow summoned a full AI session to fix it. About 60 cents and a minute of world-class intelligence, spent ticking boxes a program had already computed the exact fixes for.

🧾
The tell

The planner's output IS the fix: a list of ready-to-run commands. When the remedy is already written down, executing it is clerical work — and clerks should be free.

The new station, verbatim

This code was written by the pipeline itself (more on that in module 2). Note the two safety choices — the refusal rail and the honest ending:

CODE 

if (!line.startsWith("backlog task edit ")) throw new Error(`reconcile ${runId}: plan emitted a non-command line, refusing to execute: ${line}`);
commands.push(line);
// Honest report, no retry: a non-empty re-plan means this failure wasn't pure bookkeeping.
const planEmpty = replan.status === 0 && replan.stdout.trim() === "";
PLAIN ENGLISH

Only one shape of command may ever be executed — anything unexpected from the planner is treated as a bug and refused, not run.

Everything that passes the rail joins the to-run list.

After running the fixes, ask the planner again — and if work remains, say so instead of looping.

"Fully reconciled" is only claimed when the planner has nothing left to say.

Key Insight

Escalation ladders are how you spend intelligence well: cheapest fixer first, and each rung must honestly report what it couldn't fix rather than retry forever. The model becomes the escalation, not the default.
