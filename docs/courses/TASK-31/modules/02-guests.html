Module 2 · Guests, Not Squatters

Two invited companions and a doorbell

The lifecycle works best alongside two outside tools: Backlog.md (the task board) and GitHub Spec Kit (feature specs). Neither belongs to praxisflux — they are officially supported peers, and TASK-31 made the relationship explicit. Bootstrap never assumes them, never installs them behind your back, and never squats on a setup you already have. It rings the doorbell:

🔍
1 · Look before asking

The skill checks whether each companion's command-line tool is installed on your machine. That's the fork in the road.

💬
2 · Absent? Recommend, don't require

You're told the tool is officially supported and given the install command. Declining is fine — the project is planted without that companion's rules, and opting in later is one re-run away.

🚪
3 · Present? Ask to opt in

Having a tool installed doesn't mean this project should use it. One question per companion — and on an update, your previous answer is the remembered default.

🤝
4 · Opting in means bootstrap does the setup

It runs the companion's own initializer for you — and skips it entirely if the folder shows the companion is already set up. Your answers select which convention blocks get planted, and the stamp file remembers them.

The stamp that refuses to lie

Every bootstrapped project carries a small stamp file recording which plugin version planted it and which companions it opted into. One guard keeps the stamp honest:

CODE 

// A drifted, unconfirmed block means nothing was planted —
// don't advance the sentinel past it.
if (!check && !same && claudeMd !== "drifted") {
  writeFileSync(sentinelPath, JSON.stringify({ ...desired, plantedAt }));
}
PLAIN ENGLISH

If the rulebook refresh was refused pending your consent…

…the stamp must keep saying the OLD version is what’s planted — because it is.

Only when something really was written…

…does the stamp advance to the new version and choices…

…so the stamp never claims a state the folder isn’t actually in.

💡
Key Insight — set the table, don't cook

Bootstrap creates no wiki, no course, no lessons, and never invokes the plugins that do. It plants the rules, wires the transport, and hands off — telling you what's now possible (ground the codebase, spec the first feature) without doing any of it. In this suite, tools compose through files and gates, never by calling each other.
