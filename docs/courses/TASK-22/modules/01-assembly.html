Module 1 · The Assembly Line

Who does what, and who says so

Earlier work proved each station alone: checkers that judge work by its files, an AI that runs a skill with nobody watching, and small programs that compute routine bookkeeping. TASK-22 wired them into an actual assembly line, run by an orchestrator called n8n.

Two processes, three kinds of station

The design splits cleanly. n8n owns sequencing — which step happens next, and where to pause for a person. A tiny local service (the runner) owns execution — it has the actual tools, exposed as a handful of named steps:

⚙️
Deterministic stations — checkout, gate

Exact, repeatable programs. The gate judges work purely by the files it left behind.

🤖
The model station — agent

An unattended AI session, used only for genuinely creative work: fixing what a gate names, or writing new code.

🙋
The human station — approve

The line physically parks until a person posts their approval. No approval, no landing — by structure, not by policy.

The gate station, verbatim

Here is the runner's actual gate step. Note whose judgment it consults — a published package, the same one any outsider would download:

CODE 

const check = spawnSync("npx", ["-y", "@praxisflux/gates", "--gates", "spec-bridge", "--path", "."],
  { cwd: run.dir, encoding: "utf8", timeout: 120_000 });
const pass = planEmpty && checkPass;
return { pass, failureText: pass ? "" : [plan.stdout.trim(), check.stdout.trim()].filter(Boolean).join("\n") };
PLAIN ENGLISH

Download-and-run the project’s published checker — the exact same one anyone on the internet would get…

…pointed at this run’s working folder, with a two-minute time limit.

The verdict: the bookkeeping must be fully reconciled AND the checker must approve.

On failure, hand back the checker’s complaint word-for-word — it becomes the AI’s repair instructions.

💡
Key Insight

The failure message and the repair instructions are the same text. When your checkers write error messages a person could act on, a machine can act on them too — that one habit is what makes the retry loop free.
