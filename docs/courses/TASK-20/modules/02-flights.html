Module 2 · The Test Flights

Three runs, nobody watching

A claim like "AI can run unattended" is worthless without a test. So TASK-20 built a small throwaway project — a practice dummy with a task board and a work plan — and launched the real skill at it three times, using one command.

The launch command

This is the actual command, verbatim. One line of it does almost all the talking:

CODE 

claude -p "/spec-bridge:sync" \
  --allowedTools "Bash(node:*),Bash(backlog:*),Bash(git:*),Read,Glob,Grep" \
  --output-format json > run.json
PLAIN ENGLISH

Start an AI session with no chat window, and hand it one instruction: run the board-syncing skill.

Grant it a specific toolbox — it may run these named programs and read files, nothing we did not list.

When it finishes, save a machine-readable receipt (cost, timing, result) into a file for inspection.

The flag -p is the whole trick: it tells the CLI to run one instruction to completion and exit — no conversation, no waiting for a human.

The three flights

1
The catch-up

The board was behind the real work. Unattended, the skill moved the task forward and mirrored the plan onto it. 8 turns · 49 seconds · $0.71

2
The finish line

The work plan was regenerated and everything checked off. The skill replaced the stale plan items and moved the task to Done — with a written summary derived from the files. 6 turns · 43 seconds · $0.67

3
The honest retreat

New unfinished work appeared after Done. The skill moved the task backwards — from Done to In Progress — because the files no longer proved Done. 10 turns · 45 seconds · $0.75

💡
Key Insight

Flight three is the one that matters. Any system can march forward; a trustworthy one retreats when the evidence stops supporting its status. No human told it to — the rule is in the files.

Listen in on a run

If the three parties could talk, an unattended run would sound like this:

⏰
Scheduler

Claude, the board in this project is behind the real work. Run the sync skill. I will not be reading along.

🤖
AI Session

On it. The skill says: compute the exact edits first, then apply them, then verify my own work before I finish.

🤖
AI Session

Done — board updated, my own checks pass, and I never touched the plan files. Ending the session. Exit code: 0.

🚧
Gate

Nice speech. Inspecting the files anyway… nothing left to reconcile, every status matches its evidence. NOW it is done.

⏰
Scheduler

Verdict logged. Total: 49 seconds, 71 cents, zero humans interrupted.

Notice who has the last word before the scheduler logs anything: the gate. Every flight ended with that same inspection — silence from the checker, a clean pass from the gate, and zero stray edits outside the skill's permitted territory.
